# Chosun_OpenSW
오픈소스 SW 과제입니다.

명령어 top, ps, jobs, kill에 관한 설명을 담고 있습니다.

## top

`top` 명령어는 현재 OS의 상태를 나타내주는 CLI 어플리케이션입니다.

명령어 입력 시, 아래 사진과 같은 화면이 출력됩니다.

![image](https://github.com/SuwanKwon/Chosun_OpenSW/assets/170448896/b734db0e-971b-41b2-808c-596995495d6c)

화면에 표시되는 정보는 일반적으로 3초마다 갱신되며, `-d (숫자)` 옵션을 통해 실행 시, 갱신 주기를 변경할 수 있습니다.

### 요약 영역

화면 상단에 표시되는 영역은 요약 영역으로, 주로 전체 프로세스가 차지하고 있는 리소스를 표시합니다. 

- 첫 번째 줄에는 현재 시각, 접속 중인 유저 수, load 평균을 표시해줍니다.

- 두 번째 줄에는 수행 중인 프로세스 수에 관련된 정보로, 실행 중이거나 대기 중인 프로세스 수 등을 표시해줍니다.

- 세 번째 줄은 CPU의 사용량에 관련된 정보로, 영역별 CPU 사용 비율 등을 표시해줍니다.

- 네 번째와 다섯 번째 줄은 메모리와 관련된 정보로, 총 메모리와 사용 중인 메모리, 스왑 메모리 등을 표시해줍니다.

### 디테일 영역

요약 영역 하단에 표시되는 영역은 디테일 영역으로, 각 프로세스에 대한 내용을 표시해줍니다. 

- **PID**는 프로세스의 ID를 표시하며, 각 프로세스마다 고유한 값을 가집니다.

- **USER**은 해당 프로세스를 실행한 유저를 표시합니다.

- **PR**은 프로세스의 우선순위이며, **NI**는 프로세스의 CPU 자원 사용 우선순위로, 해당 우선순위에 영향을 주는 값입니다.

- **VIRT**는 프로세스가 차지하는 가상 메모리를, **RES**는 프로세스가 실제로 차지하는 물리 메모리를, **SHR**는 다른 프로세스와 공유하는 메모리를 표시합니다.

- **S**는 현재 프로세스의 상태를 표시합니다.

- **%CPU**는 프로세스가 사용 중인 CPU의 비율을, **%MEM**은 프로세스가 사용 중인 메모리의 비율을 표시합니다.

- <strong>TIME\+</strong>은 프로세스가 CPU를 사용한 시간을 표시합니다.

- **COMMAND**는 명령어의 이름으로, 작업을 시작할 떄 사용되는 명령어나, 프로그램의 이름을 표시합니다.

-----------------

### 명령어

`top` 명령어를 실행한 후 이용할 수 있는 커맨드가 있습니다.

- k (PID) : 프로세스를 종료시킵니다.

- M, P, N, T : 순서대로 메모리 사용량, CPU 사용량, 프로세스 ID, 실행 시간을 기준으로 정렬합니다.

- R : 오름차순 혹은 내림차순으로 정렬합니다.

- H : 프로세스 대신 쓰레드를 기준으로 표시합니다.

- 1 : CPU의 사용량을 코어별로 표시합니다.

- t, m : 순서대로 CPU 사용량과 메모리 사용량의 표시 방법을 변경합니다.

- u : 사용자에 따라 프로세스를 표시합니다.

- q : top 명령어 실행을 종료합니다.

## ps

`ps` 명령어는 현재 사용자가 실행한 프로세스 목록을 출력해주는 명령어입니다.

명령어 입력 시, 아래 사진과 같은 화면이 출력됩니다.

![image](https://github.com/SuwanKwon/Chosun_OpenSW/assets/170448896/ba9ff471-cf44-467c-ae98-09db2a8cba83)

기본적으로 프로세스 ID(PID), 터미널 번호(TTY), CPU를 사용한 시간(TIME), 프로세스의 실행 명령어(CMD)를 표시합니다.

`-f` 옵션과 `-l` 옵션을 사용하면 더 상세한 내용을 출력해줍니다.

### 풀 포맷 (-f)

아래 사진은 `-f` 옵션과 함께 명령어를 실행했을 때 출력되는 화면입니다.

![image](https://github.com/SuwanKwon/Chosun_OpenSW/assets/170448896/73292ab2-bf5c-427f-85af-e099c9c02e56)

- **UID**는 해당 프로세스를 실행한 유저를 표시합니다.

- **PID**는 프로세스의 ID를, **PPID**는 프로세스의 부모 프로세스의 ID를 표시합니다.

- **C**는 CPU의 사용량을 표시합니다.

- **STIME**은 프로세스가 시작된 시간을 표시합니다.

- **TTY**는 프로세스가 실행된 터미널의 종류와 번호를 표시합니다.

- **TIME**은 프로세스가 CPU를 사용한 시간을 표시합니다.

- **CMD**는 실행된 프로세스의 이름이나 명령어를 표시합니다.

### 긴 포맷 (-l)

아래 사진은 `-l` 옵션과 함께 명령어를 실행했을 때 출력되는 화면입니다.

![image](https://github.com/SuwanKwon/Chosun_OpenSW/assets/170448896/ca3faf4f-4825-4d8d-8b0c-f42c93f26598)

- **UID**는 해당 프로세스를 실행한 유저를 표시합니다.

- **PID**는 현재 실행 중인 프로세스의 ID를, **PPID**는 실행 중인 프로세스의 부모 프로세스의 ID를 표시합니다.

- **C**는 CPU의 사용량을 표시합니다.

- **PRI**는 프로세스의 우선순위를 표시하며, **NI**는 프로세스의 CPU 자원 사용 우선순위로, 프로세스 우선 순위 값에 영향을 줍니다.

- **ADDR**은 프로세스의 메모리 사용 주소를 표시합니다.

- **SZ**는 프로세스의 가상 메모리 사용량을 표시합니다.

- **TTY**는 프로세스가 실행된 터미널의 종류와 번호를 표시합니다.

- **TIME**은 프로세스가 CPU를 사용한 시간을 표시합니다.

- **CMD**는 실행된 프로세스의 이름이나 명령어를 표시합니다.

----------------

### 옵션

`ps`의 옵션에는 `-f`나 `-l` 외에도 이용할 수 있는 옵션이 많습니다.

- `-A` : 실행 중인 모든 프로세스를 출력합니다.

- `-e` : 커널 프로세스를 제외한 모든 프로세스를 출력합니다.

- `-M` : 64비트로 실행 중인 프로세스를 출력합니다.

- `-m` : 커널 스레드를 함께 출력합니다.

- `-p (PID)` : 특정 PID를 지정해 출력합니다.

- `-r` : 현재 실행 중인 프로세스를 출력합니다.

- `-u (UID)` : 특정 사용자의 프로세스를 출력합니다.

## jobs

`jobs` 명령어는 프로세스의 작업 상태를 출력해주는 명령어입니다.

명령어 입력 시, 아래 사진과 같은 화면이 출력됩니다.
![image](https://github.com/SuwanKwon/Chosun_OpenSW/assets/170448896/5f5dd871-3726-4a7e-8a03-fd94b6f2c7d8)

화면에서 순서대로 작업ID, 작업 상태, 실행 명령어를 표시합니다.

작업 ID 뒤에 표시된 <strong>+</strong>는 백그라운드(bg)와 포그라운드(fg) 전환 시, fg로 전환될 프로세스를 의미하며, <strong>-</strong>는 +로 표시된 프로세스가 종료될 경우, 다음으로 fg로 전환될 프로세스를 의미합니다.

### 작업 상태

작업 상태는 크게 다음과 같이 분류됩니다.

- **실행 중 (Running)** : 작업이 아직 실행 중임을 나타냅니다.

- **완료 (Done)** : 작업이 완료되었음을 나타냅니다.

- **멈춤 (Stopped)** : 작업이 일시적으로 중단되었음을 나타냅니다.

- **종료됨 (Terminated)** : 작업이 종료되었음을 나타냅니다.

- **죽었음 (Killed)** : 작업이 강제로 종료되었음을 나타냅니다.

-----------------

### 옵션

`jobs`를 사용할 때, 옵션을 통해 원하는 작업만을 출력하게 할 수 있습니다.

- `-l` : PID를 함께 출력합니다.

- `-n` : 마지막 jobs 출력 이후, 변경 사항이 있는 작업만 출력합니다.

- `-p` : 작업 내 프로세스의 PID만 출력합니다.

- `-r` : 현재 실행 중인 작업만 출력합니다.

- `-s` : 멈춘 작업만 출력합니다.

## kill

`kill` 명령어는 프로세스를 종료시키는 명령어입니다.

주로 `kill (옵션) (PID)` 형식으로 사용되며, 특정 PID에 해당하는 프로세스를 종료시킵니다.

------------------

### 옵션

`kill` 명령어의 옵션에는 시그널을 넣어 프로세스의 종료 방식을 설정할 수 있습니다.

- `-l` : 사용할 수 있는 모든 시그널을 출력합니다.

- <code>-9</code> : 프로세스를 **강제** 종료합니다.

- <code>-15</code> : 프로세스를 종료합니다. (기본값)

- <code>-18</code> : 중지된 프로세스를 재실행합니다.

- <code>-19</code> : 프로세스를 중지합니다.
